# Бот для сайта «Общажня»

Умеет: показывать профиль (созданный через бота), искать пользователей сайта по нику,
публиковать видео в общую ленту сайта от имени вашего Telegram-профиля.

## 1. Установи Python и зависимости

Нужен Python 3.10+.

```bash
cd telegram_bot
python -m venv venv
source venv/bin/activate      # на Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## 2. Получи токен бота

У [@BotFather](https://t.me/BotFather): `/mybots` → выбери бота → `API Token`.

## 3. Получи ключ сервисного аккаунта Firebase

Это отдельный секретный файл, который даёт боту доступ к базе данных сайта
(в обход обычных правил безопасности — как у администратора).

1. Заходишь в [Firebase Console](https://console.firebase.google.com/) → твой проект (`guestbook-site-5b1ab`)
2. ⚙️ **Настройки проекта → Service accounts (Сервисные аккаунты)**
3. **Generate new private key** → скачается `.json` файл
4. Положи его в папку `telegram_bot/` и назови, например, `service-account.json`

⚠️ **Этот файл нельзя выкладывать в открытый доступ, публичный GitHub-репозиторий
или кому-либо пересылать** — с ним можно полностью читать/менять/удалять базу данных сайта.

## 4. Настрой переменные окружения

Скопируй `.env.example` в `.env` и заполни своими значениями:

```bash
cp .env.example .env
```

Открой `.env` и впиши:
- `BOT_TOKEN` — токен из шага 2
- `FIREBASE_CREDENTIALS_PATH` — путь к файлу из шага 3 (например `./service-account.json`)
- `FIREBASE_STORAGE_BUCKET` — уже подставлен `guestbook-site-5b1ab.firebasestorage.app` (как в коде сайта)
- `SITE_URL` — ссылка на твой сайт (необязательно)

## 5. Запусти бота

```bash
python bot.py
```

Пока эта команда выполняется — бот работает. Как только закроешь терминал/выключишь
компьютер — бот перестанет отвечать. Для круглосуточной работы бота его нужно
разместить на сервере, который работает постоянно (см. ниже).

## Как это устроено

- **👤 Мой профиль** — у пользователей, которые публикуют видео через бота, автоматически
  создаётся отдельная запись в коллекции `telegram_users` в Firestore (имя, ник, аватар,
  число публикаций). Это НЕ то же самое, что аккаунт на сайте, зарегистрированный по email —
  это отдельный "гостевой" профиль на основе Telegram.
- **🔍 Найти пользователя** — ищет среди **зарегистрированных на сайте** аккаунтов
  (коллекция `users`), которые заводятся при регистрации email+паролем. Поиск ищет
  по совпадению начала ника и чувствителен к регистру букв — это ограничение
  самого Firestore (он не умеет искать "содержит подстроку" без сторонних сервисов
  вроде Algolia).
- **🎥 Опубликовать видео** — видео скачивается из Telegram, заливается в Firebase
  Storage и становится публично доступным по ссылке (`make_public()`), затем создаётся
  запись в коллекции `messages` — точно в том же формате, что использует сам сайт при
  публикации поста. Поэтому видео сразу же появляется в общей ленте на сайте.

## Постоянная работа бота (хостинг)

Варианты, где бот может работать 24/7 бесплатно или недорого:
- **Railway.app** / **Render.com** — background worker, поддерживают `requirements.txt` из коробки
- Обычный недорогой VPS (запускать через `systemd` или `screen`/`tmux`, чтобы бот жил
  после закрытия терминала)

На всех вариантах не забудь загрузить `.env` и `service-account.json` тем же способом
(как переменные окружения/секреты хостинга), а не в открытом виде в репозиторий.
